Procédure de conversion ePub vers XML TEI¶

Note

Seule cette section est en français pour aider l’opérateur à faciliter sa tâche. Par contre, la documentation des codes se fait en anglais.

See also

Liste des abréviations

Préparation avant la conversion¶

Pour la conversion, quelques étapes de préparation sont nécessaires pour pouvoir convertir le fichier ePub en fichier XML TEI.

Procédure de balisage¶

La balisage des attributs nécessaires pour la conversion est de la forme xml-<tag_name>.

Ici comme exemple, nous spécifions que toutes les <p class="Ct"> sont à convertir en <caption> :

Ceci :

<p class="Ct xml-caption">Cupidatat occaecat sint consectetur labore.</p>

Donne :

<caption rendition="justify">Cupidatat occaecat sint consectetur labore.</caption>

Note

Pas besoin de préciser à chaque fois le même type de tag pour chaque style. Une fois suffit.

Important

Les feuilles de style (classes) principaux doivent être au début de chaque déclaration.

Exemple :

Comme ceci :

<p class="Ct xml-caption">Cupidatat occaecat sint consectetur labore.</p>

Et non :

<p class="xml-caption Ct">Cupidatat occaecat sint consectetur labore.</p>

Attention

La balisage ne peut pas s’effectuée seule. Les balises XML doivent toujours être accompagnés de feuille de style (ou classe). Donc, seules les balises ayant des feuilles de style (ou classes) peuvent être balisées.

En-tête¶

Part¶

Balisée par xml-part, elle sera convertie en <head/>.

Ce code :

<p class="Chapter xml-chapter">Cupidatat occaecat sint consectetur labore.</p>

Donne :

<head>Cupidatat occaecat sint consectetur labore.</head>

Chapter¶

Balisé par xml-chapter, elle sera convertie en <head rendition="#chaptertitle"/>.

Ce code :

<p class="Chapter xml-chapter">Cupidatat occaecat sint consectetur labore.</p>

Donne :

<head type="#chaptertitle">Cupidatat occaecat sint consectetur labore.</head>

Section¶

Balisée par xml-stitre1, elle sera convertie en <head type="#sectiontitle"/>.

Ce code :

<p class="Stitre1 xml-stitre1">Cupidatat occaecat sint consectetur labore.</p>

Donne :

<head type="#sectiontitle">Cupidatat occaecat sint consectetur labore.</head>

Note

Sous-section, sous-sous-section et le reste suivent le même principe.

Éléments à supprimer¶

xSpan¶

Balisée par xml-xspan, elle sera juste effacée mais le contenu reste.

Ce code :

<p class="Ct xml-caption">Cupidatat <span class="charOverride-10 xml-xspan">occaecat sint consectetur</span> labore.</p>

Donne :

<caption>Cupidatat occaecat sint consectetur labore.</caption>

Note

Le x avant le nom de la balise (xml-xhr, xml-xspan, ...) sert de repère pour indiquer que cette balise sera juste effacée en gardant son contenu par contre.

xHr¶

Balisée par xml-xhr, elle sera juste effacée.

Exemple :

<hr class="FootnoteRule xml-xhr"/>

Liste des balises¶

En-têtes¶

  • xml-part : Titre partie. (Voir Part)
  • xml-chapter : Titre chapitre. (Voir Chapter)
  • xml-stitre1 : Titre section. (Voir Section)
  • xml-stitre2 : Titre sous-section. (Voir Section note)

Images¶

  • xml-fig : Figure. (Voir Images_)
  • xml-fig-div : Bloc d’image.
  • xml-caption : Légende. (Voir Caption)

Tableaux¶

  • xml-table : Tableau.
  • xml-table-head : Titre de tableau.

Notes¶

Notes de bas de page¶

  • xml-footnote : Note de bas de page.
  • xml-footnote-link : Lien note de bas de page.
  • xml-footnote-div : Bloc note de bas de page (ID).

Note de fin¶

  • xml-endnote : Note de fin de chapitre.
  • xml-endnote-link : Lien note de fin de page.
  • xml-endnote-div : Bloc note de fin de page (ID).

Appelle de note¶

  • xml-appnote-a : Appel de note de type a.
  • xml-appnote-span : Appel de note de type span.
  • xml-appnote-link : Lien appel de note.

Blocs standards¶

  • xml-para-std : Texte standard.
  • xml-list : Liste.

Elements à supprimer¶

  • xml-xspan : Span qui sera supprimé lors de la conversion. (Voir xSpan)
  • xml-xhr : Filet de note qui sera effacé lors de la conversion. (Voir xHr)

Note

D’autres balises peuvent ne pas être dans la liste car elles varient d’un fichier à un autre.

Fonctionnalités¶

Ce que le programme peut faire¶

  1. Convertir les attributs suivant ses balises réspéctives.

    Donc les attributs sont convertis en leurs équivalences sur XML dans cet ordre précis :

    <hi rend="sc | up | lo > u > b > i > sup | sub"/>

  2. Convertir quelques balises suivant la structure TEI. (Voir Liste des balises)

En cours de concéption¶

  1. Monter les notes de bas de pages et de fins.
  2. Automatisation des blocs de division pour les niveaux de titre et sous-titre.
  3. Création d’une interface graphique.
